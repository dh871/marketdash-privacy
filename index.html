Privacy Policy

MarketDash (Finance Dashboard)

Effective date: February 21, 2026

Last updated: February 21, 2026

1. Who We Are

This Privacy Policy describes how MarketDash ("App") handles information.

Owner: Rohan Dalal
Support: Use TestFlight feedback or the support contact listed in App Store Connect.

2. Plain-Language Summary

No account required No ads No third-party analytics SDK API keys stored in iOS/macOS Keychain User BYOK model (you bring your own keys) No developer-run backend for your app data

We do not sell your personal data. The App is primarily on-device. When you use live market data and AI features, your device calls the selected third-party APIs directly.

3. Information Processed

3.1 Data you enter

  • Ticker symbols, watchlists, portfolio weights, screener inputs, and other tool inputs.
  • Optional API keys you paste (FMP, Finnhub, Twelve Data, OpenAI, Anthropic, Gemini).
  • Optional AI prompts/questions you type in the Cross-Tool AI Copilot and related AI tools.

3.2 Data fetched from providers

  • Market, news, filings, and broker-flow data from your enabled market data providers.
  • AI responses from your selected LLM provider.

3.3 Data we do not intentionally collect

  • No account profile data (name, phone number, postal address) is required for app usage.
  • No ad tracking identifiers are collected by us.
  • No third-party analytics/ads SDK is used in normal app runtime.

4. Where Data Is Stored

4.1 API keys

  • Primary storage: Native Keychain on your device.
  • Security class: Keychain items are stored with kSecAttrAccessibleAfterFirstUnlock.
  • Migration behavior: legacy web-storage key aliases are read once at startup, copied into Keychain flow, then removed.
  • Fail-closed behavior: if native persistence is temporarily unavailable, key is session-active only until queued persistence succeeds.

4.2 Other app state

Non-key app state may be stored locally (WebView local storage/UserDefaults), including:

  • Watchlists, portfolio snapshots, preferences (such as theme), and tool UI state.
  • Cross-tool context snapshots used for AI continuity.
  • Limited AI memory records (for follow-up continuity), including your question text and generated summary.

5. What Leaves Your Device

The following can be transmitted from your device to third-party APIs to fulfill your request:

  • Market data calls: ticker symbols, query parameters, and your provider API key/token.
  • LLM calls: your prompt plus selected context (for example: active ticker(s), recent quote/news/filings summaries, and limited follow-up context).
  • Network metadata: destination provider receives standard request metadata such as IP address and user-agent/device-network metadata that accompanies HTTPS requests.

If you use AI features, those inputs are processed by the selected LLM provider under that provider's terms and privacy policy.

We do not operate a custom server that receives and stores your prompts, keys, or portfolio data for normal app operation.

6. Third-Party Services Used

  • Finnhub
  • Financial Modeling Prep (FMP)
  • Twelve Data
  • OpenAI
  • Anthropic
  • Google Gemini API

Requests are restricted in-app by an HTTPS domain allowlist policy. Public CORS relay/proxy services are not used for market data in compliance mode.

7. How We Use Data

  • Provide finance tool functionality and live data views.
  • Maintain local user state and preferences.
  • Generate AI outputs only when AI features are explicitly used.
  • Provider requests are initiated by your in-app actions (for example: Refresh, Analyze, Scan, Send in AI Copilot).
  • Enforce local AI budgeting/rate controls and show usage spend in app.

8. Sharing and Selling

  • We do not sell personal information.
  • We do not broker or monetize user data.
  • Data is shared only with the external provider(s) needed to perform your requested feature.
  • Because API requests go directly to your selected providers, those providers process data under their own terms and privacy policies.
  • Any third party receiving app data is expected to provide protection of user data consistent with its privacy policy and applicable law.

9. Retention

  • API keys remain in Keychain until you clear them.
  • Local tool state/context remains on-device until cleared by app actions or uninstall.
  • AI session turns are in-session memory; limited AI memory/context snapshots may persist locally for continuity.
  • Third-party provider retention is governed by each provider's own policy.

We cannot directly delete data retained by third-party providers. To delete provider-side data, use that provider account's controls/support channels.

10. Security

  • API keys use native Keychain storage path.
  • Network access is constrained by host allowlist and HTTPS policy in app runtime.
  • No security measure is perfect; protect your device and API credentials.

11. Children's Privacy

The App is not directed to children under 13, and we do not knowingly collect personal information from children under 13.

12. Your Choices and Rights

  • Clear keys from Settings at any time.
  • Delete local app data by using in-app clear functions or uninstalling the app.
  • Choose whether to enable AI and which provider key to use.
  • To stop AI processing entirely, clear OpenAI/Anthropic/Gemini keys and do not use AI features.
  • To stop market-provider processing, clear Finnhub/FMP/Twelve keys and do not run live data tools.

Third-party API providers may process requests in regions where they operate.
